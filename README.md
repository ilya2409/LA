# 📊 Log Analyzer (LA)

**CLI + Web** инструмент для анализа лог-файлов с фильтрацией, статистикой, AI‑сводкой и историей анализов.

![Python](https://img.shields.io/badge/python-3.12-blue)
![FastAPI](https://img.shields.io/badge/FastAPI-0.115-green)
![Ollama](https://img.shields.io/badge/Ollama-gemma3:1b-orange)

---

## ✨ Возможности

- **Загрузка лог-файлов** (до 10 МБ, форматы `.log`, `.txt`)
- **Фильтрация** по уровню (DEBUG, INFO, WARNING, ERROR, CRITICAL), ключевому слову, диапазону дат
- **Статистика** – распределение по уровням, топ-3 самых частых сообщений, временной диапазон
- **Локальная AI‑сводка** – анализ лога через Ollama (модель `gemma3:1b`) с понятными рекомендациями
- **История анализов** – последние 10 результатов сохраняются в браузере, экспорт в `.txt`
- **Тёмная / светлая тема** – сохраняется в localStorage
- **Защита** – rate limiting (1 запрос в 10 секунд), проверка типов файлов, защита от path traversal
- **CLI‑версия** – для использования в терминале (парсинг, фильтры, статистика)

---

## 🏗 Архитектура

```
Браузер → Caddy (reverse proxy, опционально) → FastAPI (uvicorn) → Парсер логов → Ollama (AI)
                                                ↓
                                          localStorage (история)
```

**Технологии:**
- Backend: Python 3.12, FastAPI, Uvicorn, Gunicorn
- Frontend: HTML5, CSS3 (кастомный GitHub‑стиль), Vanilla JS
- AI: Ollama, модель `gemma3:1b` (работает на CPU)
- Безопасность: slowapi (rate limiting), экранирование вывода
- CLI: стандартная библиотека (argparse, re, datetime)

## 🚀 Быстрый старт

### Установка и запуск (локально)

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/yourusername/log-analyzer.git
   cd log-analyzer
   ```

2. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```

3. **Установите и запустите Ollama**
   - Скачайте [Ollama](https://ollama.com) и установите
   - Загрузите модель:
     ```bash
     ollama pull gemma3:1b
     ```
   - Убедитесь, что Ollama запущен: `ollama serve`

4. **Запустите веб-сервер**
   ```bash
   python run_web.py
   ```
   Сервер будет доступен по адресу `http://localhost:8000`

### Запуск CLI‑версии
```bash
python main.py logs/sample.log --stats
python main.py logs/sample.log --level ERROR --search "timeout"
```

## 🖥 Веб-интерфейс

### Основные элементы
- **Форма загрузки** – выбор файла, настройка фильтров, чекбокс «Только статистика»
- **Результат** – отображается либо список записей (с подсветкой уровней), либо статистика
- **ИИ сводка** – появляется после анализа (если не выбран режим статистики)
- **История** – автоматически сохраняется, выгружается кнопкой «Сохранить историю»
- **Тёмная тема** – переключатель в правом верхнем углу (сохраняется)

### Пример работы

1. Загрузите `sample.log`
2. Выберите уровень `ERROR` → покажутся только ошибки и критические события
3. Нажмите «Анализировать» → увидите результат и через 1‑2 секунды – AI‑сводку
4. Нажмите «Сохранить историю» → скачается `.txt` с последними 10 анализами

## 🐳 Деплой (опционально, через Docker)

См. файлы `Dockerfile` и `docker-compose.yml` в репозитории.

Кратко:
```bash
docker compose up -d --build
```
После этого сервис будет доступен на порту `8001` (локально) или через reverse proxy (Caddy, Nginx).
